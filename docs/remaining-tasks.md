# Remaining Tasks

このメモは、現時点の実装状況を基準に「完了済み」「残っているもの」「既知制約」を分けて整理したものです。

## Completed

- `GET /api/items` は一覧取得に責務を限定し、タグ集計は `GET /api/tags` に分離済みです。
- タグ絞り込み、検索、並び順、表示範囲、ページ番号は URL クエリと同期済みです。
- `items` 一覧のページネーションは実装済みです。
- `App.tsx` は認証、アカウント、アイテム編集、一覧表示をコンポーネントと hooks に分割済みです。
- タグの正規化と重複除去はバックエンドとフロントの両方で行っていますが、最終的な正はバックエンドです。
- 「自分のアイテムだけ編集・削除できる」権限制御はバックエンドで強制し、フロントは追従する UI 制御を持っています。

## Remaining

- `GET /api/tags` を使ったタグ一覧の見せ方は改善を進めていますが、まだ「もっと見る」や並び替え切替のような拡張余地があります。
- `frontend/src/App.tsx` 以外の周辺 UI も含めて、認証系とアイテム系の責務分離をさらに進める余地があります。
- Alembic migration の運用ルールを整備し、スキーマ変更時の手順を `backend/README.md` に詳しく残す必要があります。

## Known Constraints

- `frontend` の `npm run build` は、この実行環境では Vite の変換自体は進むものの、終了コードだけ `1` で返ることがあります。
- `backend` の `pytest` 実行時に `.pytest_cache` 系の権限警告が出ますが、テスト自体は通っています。
- 実装が進んだことで `App.tsx` の肥大化はかなり解消しましたが、今後さらに分割すると保守性は上がります。

## Suggested Follow-ups

- タグ一覧のソートや表示ルールを詰める。
- マイグレーション手順とリリース時の注意点を `backend/README.md` に追記する。
- 認証周りと一覧周りの UI を、必要ならもう一段階コンテナ分離する。

# Remaining Tasks

## Known Constraints

- `frontend` の `npm run build` は、この実行環境では Vite の変換自体は進むものの、終了コードだけ `1` で返ることがあります。
- `backend` の `pytest` 実行時に `.pytest_cache` 系の権限警告が出ますが、テスト自体は通っています。
- `frontend/src/App.tsx` は責務を減らしたものの、認証・フォーム送信・一覧状態をまだ 1 ファイルで持っているため、今後は `auth` と `items` のコンテナ分離余地があります。

## Next Candidates

- タグ絞り込み状態を URL クエリに同期して、再読み込みや共有時にも同じ一覧条件を復元できるようにする。
- `GET /api/tags` を使って人気タグ順や未使用タグ除外など、タグ一覧の見せ方を改善する。
- `items` 一覧にページネーションか件数制限を入れて、データ量増加時のレスポンスと UI 負荷を抑える。
- `frontend` 側の `App.tsx` をさらに分割し、認証まわりとアイテム管理まわりを別コンポーネントへ切り出す。
- Alembic migration に運用ルールを追加し、スキーマ変更時の手順を `backend/README.md` に詳しく残す。

## Current Design Notes

- `GET /api/items` は一覧取得に責務を限定しており、タグ集計は `GET /api/tags` に分離済みです。
- タグの正規化や重複除去はバックエンドとフロントの両方で行っていますが、最終的な正はバックエンドです。
- 「自分のアイテムだけ編集・削除できる」権限制御はバックエンドで強制し、フロントはそれに追従する UI 制御だけを持っています。

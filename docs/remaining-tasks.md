# Remaining Tasks

このメモは、現時点の実装状況を基準に「完了済み」「残っているもの」「既知制約」を分けて整理したものです。

## Completed

- `GET /api/items` は一覧取得に責務を限定し、タグ集計は `GET /api/tags` に分離済みです。
- タグ絞り込み、検索、並び順、表示範囲、ページ番号は URL クエリと同期済みです。
- `items` 一覧のページネーションは実装済みです。
- タグ一覧は上位件数表示と「もっと見る」に対応済みです。
- `App.tsx` は認証系とアイテム系のセクションに分割され、各セクション配下でコンポーネントと hooks に整理済みです。
- タグの正規化と重複除去はバックエンドとフロントの両方で行っていますが、最終的な正はバックエンドです。
- 「自分のアイテムだけ編集・削除できる」権限制御はバックエンドで強制し、フロントは追従する UI 制御を持っています。

## Remaining

- `GET /api/tags` を使ったタグ一覧の見せ方は改善を進めていますが、まだ並び替え切替や表示ルールの追加調整余地があります。
- 認証系とアイテム系の内部 hooks は整理が進みましたが、将来的にはセクション単位の state 受け渡しをさらに薄くする余地があります。
- Alembic migration の運用ルールは追記済みですが、将来 migration が増えたら README 側の一覧も追随更新する必要があります。

## Known Constraints

- `frontend` の `npm run build` は、この実行環境では Vite の変換自体は進むものの、終了コードだけ `1` で返ることがあります。
- OneDrive 配下で `pytest` のキャッシュ一時ディレクトリが壊れることがあったため、現在は backend 側で `cacheprovider` を無効化しています。
- 実装が進んだことで `App.tsx` の肥大化はかなり解消しましたが、今後さらに表示コンテナや state 境界を詰める余地はあります。

## Suggested Follow-ups

- タグ一覧のソートや表示ルールを詰める。
- migration が増えた時点で `backend/README.md` の一覧を更新する。
- 認証周りと一覧周りの state 境界を、必要ならもう一段階整理する。
